# Codebook

Variable definitions for the generated files in this directory. This file is written by the pipeline scripts; edit the definitions in the generating script, not here.

## O*NET reference tables

Generated by `pipeline/convert_onet.py`.

### `task_statements.csv`

O*NET 20.1 Task Statements workbook converted to CSV with columns unchanged.

| Variable | Definition |
| --- | --- |
| `O*NET-SOC Code` | Formatted O*NET-SOC occupation code (eight digits plus punctuation, e.g. 11-1011.00). |
| `Title` | O*NET occupation title. |
| `Task ID` | O*NET task identifier. |
| `Task` | Task statement text. |
| `Task Type` | Core or Supplemental; blank when the task is unclassified. |
| `Incumbents Responding` | Number of survey incumbents rating the task. |
| `Date` | Task update date (mm/YYYY). |
| `Domain Source` | O*NET data collection source for the task. |

### `soc_2010_to_2018_crosswalk.csv`

BLS SOC 2010 to SOC 2018 crosswalk workbook converted to CSV. Contains one stray repeated-header row, which pipeline/build_crosswalk.py drops.

| Variable | Definition |
| --- | --- |
| `soc_2010` | Six-digit SOC 2010 occupation code. |
| `title_2010` | SOC 2010 occupation title. |
| `soc_2018` | Six-digit SOC 2018 occupation code. |
| `title_2018` | SOC 2018 occupation title. |

## SOC 2010 to 2018 crosswalk artifacts

Generated by `pipeline/build_crosswalk.py`.

### `soc_crosswalk_edges.csv`

Deduplicated direct crosswalk edges, one row per SOC 2010 to SOC 2018 mapping pair.

| Variable | Definition |
| --- | --- |
| `soc_2010` | Six-digit SOC 2010 occupation code. |
| `title_2010` | SOC 2010 occupation title. |
| `soc_2018` | Six-digit SOC 2018 occupation code. |
| `title_2018` | SOC 2018 occupation title. |

### `soc_2010_to_group.csv`

One row per SOC 2010 code with its crosswalk group.

| Variable | Definition |
| --- | --- |
| `soc_2010` | Six-digit SOC 2010 occupation code. |
| `title_2010` | SOC 2010 occupation title. |
| `group_id` | Connected-component ID of the bipartite SOC 2010<->2018 crosswalk graph; each component can be one-to-one, one-to-many, many-to-one, or many-to-many. |

### `soc_2018_to_group.csv`

One row per SOC 2018 code with its crosswalk group.

| Variable | Definition |
| --- | --- |
| `soc_2018` | Six-digit SOC 2018 occupation code. |
| `title_2018` | SOC 2018 occupation title. |
| `group_id` | Connected-component ID of the bipartite SOC 2010<->2018 crosswalk graph; each component can be one-to-one, one-to-many, many-to-one, or many-to-many. |
